# Lakehouse  Health Care Data Model

This document outlines a canonical data model for the  health Care business, designed for a Lakehouse architecture. The model is structured into business-aligned domains and denormalized data products, optimized for analytical workloads.

## Output Folder Structure

All artifacts for version **v1** are organized as follows:

```
v1/
  schemas/          DDL SQL files (domains, products, foreign keys, tags)
  metrics/          Metric view SQL files (one per domain)
  samples/          Sample data CSV files (one per data product)
  docs/             Excel workbook, model CSV, data model JSON, DBML schema
  vibes/            Current & next vibes context, AI logs
  ontology/         RDF/Turtle ontology schema
  readme.md         This file
  releasenotes.txt  Version release notes with change summary
```
